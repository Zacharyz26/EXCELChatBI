# 数据画像安全策略（三层数据边界）

> 状态：规范 · 适用：所有涉及"把数据交给模型"的分析切片
> 原则：**场景无关**（不假设行业、不写死业务列名）、**配置驱动**、**默认宽松、按需收紧**、**默认可用**（无配置也安全）。

本文件定义 ChatBI 中"原始数据 / 模型"之间的安全边界，是红线1 的落地规范。任何新分析能力都必须在此边界内工作。

## 1. 三层数据边界

```
第1层  模型只看画像   ：Schema + 统计摘要 + 脱敏后的采样。原始明细行绝不进模型。
第2层  引擎本地执行   ：按模型规划在本地（pandas / DuckDB）聚合，数据不出环境。
第3层  模型看聚合结果 ：只看最终聚合数字做解读，不看明细；且小分组聚合受保护。
```

- 第1层实现：`packages/governance/redaction.py`（在 `excel_parser.infer_schema` 施加）。
- 第2层实现：`packages/common/dataset_store.py::aggregate`（聚合下推到 DuckDB）。
- 第3层实现：`packages/governance/aggregation_guard.py`（`chart.gen_chart` 出图前施加）。

模型只提供"图表类型 + 列映射"，**所有数值由第2层基于真实数据计算**（红线2）。

## 2. 泄露路径与对策

红线1 不仅是"别把整表塞给模型"，还需堵住两条隐蔽路径：

| 路径 | 说明 | 对策 |
|------|------|------|
| 样本行 | 画像里的样本明细行含真实单元格 | 仅 `VALUES` 列的单元格进入样本行，其余遮蔽/置空 |
| 列级采样 | 每列 `sample_values` 是真实去重值，低基数列会暴露内容 | 按级别 / 列规则决定是否给样本值、是否打码 |
| 小分组聚合 | groupby 分组只有极少行时，聚合值≈原始明细 | 低于阈值的分组合并到"其他"或丢弃 |

## 3. 数据集安全策略结构

一个数据集的生效策略由三层合并（越具体越优先）：

```
内置默认(宽松) ⊕ config/data_policy.yaml 的 default ⊕ 数据集 sidecar 元数据(上传可传入)
```

### 3.1 敏感级别（`SensitivityLevel`，可扩展）

| 级别 | 语义（默认行为） |
|------|------------------|
| `open`（默认） | 数值/布尔/时间列给值；文本列 distinct ≤ 阈值(默认20) 给样本值，否则只给统计 |
| `internal` | 同 open，但低基数阈值更小(默认8) → 更多文本列不给样本值 |
| `restricted` | 文本列样本值一律打码；时间列只给统计；仅数值列给值 |

低基数阈值按级别配置（`low_cardinality_max_distinct`），级别越严阈值越小。

### 3.2 列级规则（`ColumnRule`，显式覆盖类型默认）

| 规则 | 效果 |
|------|------|
| `normal` | 用按类型 / 级别的默认行为 |
| `mask` | 样本值打码，**保留**统计摘要（min/max/mean 等） |
| `exclude` | 不给样本值，也**不给**统计摘要，仅留 schema（名/类型/空值率/distinct） |

### 3.3 采样动作（内部）

每列最终解析为一个动作，决定样本值与样本行单元格：

| 动作 | 样本值 | 统计摘要 | 样本行单元格 |
|------|--------|----------|--------------|
| `VALUES` | 真实值 | 保留 | 真实值 |
| `STATS_ONLY` | 无 | 保留 | 遮蔽 |
| `MASK` | 打码 | 保留 | 打码 |
| `EXCLUDE` | 无 | 无 | 置空 |

> 只有 `VALUES` 列的真实单元格会出现在发给模型的 payload 中。

## 4. 小分组保护（第3层）

- 分组样本量 `< small_group_min_size`（默认 5）视为小分组。
- `merge`（默认）：小分组合并为一个 `other_label`（默认"其他"）；
  - `sum` 保持总量守恒；`mean` 用样本量加权；`count` 累加。
  - 合并后若仍 `< 阈值`，整体丢弃，避免"其他"本身泄露单个小组。
- `drop`：直接丢弃小分组。
- 阈值与模式可按数据集覆盖。`agg=none`（不聚合）与散点图不适用小分组保护。

## 5. 可配置点一览

| 配置 | 位置 | 默认 |
|------|------|------|
| 默认级别 | `data_policy.yaml: default.level` | `open` |
| 低基数阈值 | `default.low_cardinality_max_distinct.{open,internal,restricted}` | 20 / 8 / 0 |
| 小分组阈值 | `default.small_group_min_size` | 5 |
| 小分组模式 | `default.small_group_mode` | `merge` |
| 合并桶名 | `default.other_label` | 其他 |
| 掩码标记 | `default.mask_token` | `***` |
| 数据集覆盖 | 上传 `policy` 参数 / 数据集 sidecar 元数据 | 无（用默认） |
| 脱敏实现 | `Redactor` 接口（可替换：打码/假名化/格式保留） | `DefaultRedactor`（打码） |

## 6. 已知取舍与后续

- **默认宽松的代价**：默认 `open` 下，低基数文本列（含可能敏感的内容）会给样本值。要保护，就把该列配 `mask`/`exclude`，或把数据集设为 `restricted`。这是"默认可用、按需收紧"的有意取舍。
- **本次完成**：策略结构 + 默认 + 配置加载 + 合并；采样脱敏（样本值 + 样本行）；小分组保护；聚合下推 DuckDB（类目图真实可用）。
- **留后续**：超大表流式/分块与全量统计下推；散点图大表采样下推；假名化 / 格式保留脱敏器；外部数据目录 / 多租户策略来源。
