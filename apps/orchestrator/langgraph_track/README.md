# langgraph_track（历史占位目录）

该目录来自早期“A/B 双轨”设计，当前没有运行时代码，不能作为现行复杂任务入口。

自 2026-07-21 起，复杂多步、动态重规划、完成验证和恢复已经纳入 v2.4 Agent 控制面。默认路线是在现有编排上建立统一的类型化状态机，简单快速路径、模板路径和开放 LLM 规划路径都输出同一种 TaskPlan。

是否使用 LangGraph 不是阶段 0 的预设结论。只有在 Planner/Verifier spike 完成后，Checkpoint、分支、人工介入和恢复复杂度证明现有状态机不合适，并且评测显示框架迁移有明确收益时，才启用本目录或迁移运行时。

现行依据：[`docs/Agent自主化开发规划.md`](../../../docs/Agent自主化开发规划.md)；控制面状态机细化见
[`docs/v2.4/控制面与持久化设计.md`](../../../docs/v2.4/控制面与持久化设计.md)。MCP 与 Docker 是独立横向交付轨，不构成引入
LangGraph 的理由，详见 [`docs/v2.4/MCP与Docker架构决策.md`](../../../docs/v2.4/MCP与Docker架构决策.md)。
